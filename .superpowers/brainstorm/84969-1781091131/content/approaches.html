3D Token City — Implementation Approaches

Each builds the isometric city effect differently. Pick the one that fits your goals.

A — Canvas (Recommended)

Custom Svelte canvas component. Draws isometric buildings frame-by-frame. Hover → highlight + tooltip. Consistent with existing AuraVisualizer pattern in this codebase. No new dependencies.

Pros

  • Zero new deps
  • Full pixel control
  • Smooth animated entrance
  • Matches existing code style

Cons

  • ~150 lines of canvas math
  • Accessibility requires extra work

B — CSS 3D Transforms

Pure CSS rotateX + rotateZ on a flex bar chart. Looks 3D-ish with no canvas. Simplest to build, easiest to style with Tailwind.

Pros

  • ~30 lines of Svelte
  • Fully accessible (DOM)
  • Easy color theming

Cons

  • Not true isometric — perspective distortion
  • No side faces on buildings
  • Less impressive than screenshot

Three.js / WebGL
Real 3D engine
~280 KB gzipped

C — Three.js (WebGL)

Full 3D engine. Real perspective camera, lighting, shadows. Closest to a game engine. Can rotate/zoom. Massive overkill for a dashboard widget but most impressive ceiling.

Pros

  • True 3D + orbit controls
  • Lighting/reflections
  • Unlimited visual depth

Cons

  • +280 KB bundle hit
  • Complex integration with Svelte
  • Major overkill for a chart
